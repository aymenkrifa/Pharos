Read the beacon

The beacon watches both usage windows and always shows the one closest to its limit. Both its shape and its color change, so the warning reads even in grayscale. Open the menu for exact percentages and when each window resets.

Install

Needs GNOME Shell 46, 47 or 48, and Claude Code signed in on this machine — Pharos reads the sign-in Claude Code already has.

  1. Copy Pharos into place

    Open a terminal and paste this — it downloads Pharos and puts it where GNOME looks for extensions.

    git clone https://github.com/aymenkrifa/Pharos.git
cd Pharos
glib-compile-schemas schemas/
EXT=~/.local/share/gnome-shell/extensions/pharos@aymenkrifa.github.io
mkdir -p "$EXT"
cp -r extension.js prefs.js lib metadata.json stylesheet.css schemas assets "$EXT/"
  2. Reload GNOME Shell

    On X11: press Alt + F2, type r, press Enter.
    On Wayland: log out and back in.

  3. Light the beacon

    Turn Pharos on — once is enough:

    gnome-extensions enable pharos@aymenkrifa.github.io

    …or flip the switch in the Extensions app. The tower appears in your top panel right away.

More than one account

Work and personal? Open Settings → Accounts from the Pharos menu and add each one — a label and the path to its credentials file. The menu grows a switcher, and the panel shows which account you're watching (full name, a single letter, or nothing — your choice).

One account needs no setup at all: Pharos finds Claude Code's standard sign-in on its own and keeps it fresh.

Good to know

Pharos reads Claude Code's sign-in from your home folder and talks only to Anthropic's own servers — the usage endpoint, and the same sign-in renewal Claude Code itself uses. Nothing about your usage leaves your machine for anywhere else. It's free software, GPL-2.0-or-later — the source is a short read.

If the menu ever says "Run Claude Code to re-auth" — your stored sign-in has fully expired. Open Claude Code, sign in once, and Pharos carries on at the next refresh.
